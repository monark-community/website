# Vote du Registre foncier

Construisez un système de vote où les propriétaires fonciers et/ou les locataires de biens immobiliers peuvent voter sur des propositions relatives à leur région. Liez les droits de vote à la propriété foncière (réelle ou simulée), vérifiez les votes par les signatures de portefeuille, et affichez les résultats sur une carte ou un tableau de bord. Explorez les moyens de synchroniser les votes en chaîne avec les registres hors chaîne pour une intégration civique potentielle.

![Maquette du Vote du Registre foncier](/images/project/land-registry-voting.jpg)

## Description Générale

[LandVote](https://landvote.monark.io) est une solution blockchain axée sur le civisme qui permet aux propriétaires fonciers de participer à la gouvernance communautaire liée à leurs parcelles physiques. En liant les identités des portefeuilles aux possessions immobilières vérifiées (simulées pour ce projet), la plateforme accorde des droits de vote sur les propositions relatives à l'utilisation des terres, telles que les modifications de zonage, la planification des infrastructures, ou l'allocation des ressources publiques.

Chaque vote est enregistré de manière immuable sur la blockchain et les propositions sont visibles publiquement pour plus de transparence. Le système simule les processus de prise de décision réels des villes, permettant aux propriétaires fonciers de s'engager dans des problèmes qui affectent directement leurs quartiers ou leurs biens. Les rôles comprennent le proposant, le validateur (par exemple, le représentant municipal), et l'électeur éligible basé sur la terre.

Le projet introduit également des fonctionnalités de cartographie. Les parcelles de terrain sont représentées sur une interface cartographique visuelle (par exemple, via OpenStreetMap), permettant aux utilisateurs de voir les propositions pertinentes pour leur région. Avec le temps, ce projet peut évoluer en un système complet d'audience publique numérique et de budgétisation participative.

Les étudiants qui travaillent sur LandVote combineront des modèles de gouvernance en chaîne avec des outils de cartographie, des simulations de registres immobiliers, et le design UX du bien public. Ils exploreront les systèmes d'attestation d'identité, le vote pondéré, et la construction de la confiance civique - des compétences alignées sur l'infrastructure civique de la prochaine génération.

## Livrables & Fonctionnalités Souhaitées

- Interface Web utilisant Next.js, Typescript, shadcn-ui, tailwindcss
- Authentification des utilisateurs
- Contrats intelligents pour la création de propositions et le vote régional
- Simulation de l'association portefeuille-terre
- Interface utilisateur de carte interactive pour les superpositions de vote régional
- Validation et logique de vote basées sur le rôle
- Historique des votes par parcelle

## Ressources Nécessaires

- Plateforme d'hébergement Web (Vercel)
- Base de données (Clickhouse, PostgreSQL ou GraphQL) et hébergement Backend (Typescript, Express)
- Bibliothèque de contrats intelligents (Solidité + OpenZeppelin)
- Testnet Blockchain
- APIs de cartographie (OpenStreetMap ou Leaflet.js)
- Registre de données factices pour la simulation de parcelles de terrain

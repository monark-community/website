## Required Ressources

Tools/softwares that are not in the possession of the students will be provided by our team.

- [GIT](https://git-scm.com/)
- [Node.js](https://nodejs.org/) version 22+
- [Visual Studio Code IDE](https://code.visualstudio.com/) and [Remix](https://remix-project.org/) if needed
- Web hosting platform ([Vercel](https://vercel.com/) or [Render](https://render.com/))
- Access to RPC provider
- [Tenderly](https://tenderly.co/) access to create virtual blockchain testnet
- Database ([ClickHouse](https://clickhouse.com/), [PostgreSQL](https://www.postgresql.org/) or [GraphQL](https://graphql.org/)) and Backend ([Express](https://expressjs.com/)) hosting ([Render](https://render.com/))
- Smart Contract Framework (Solidity) and libraries for secure contract development ([OpenZeppelin](https://www.openzeppelin.com/))
- ZK Network for confidential smart contracts, private logic execution and ZK identity proofs
    - SNARKs, STARKs, Semaphore, or other ZK primitives to handle privacy-preserving proofs
- Encrypted Off-Chain Storage Layer ([IPFS](https://ipfs.tech/) with [Filecoin](https://filecoin.io/)) for encrypted data storage and controlled access
- Realistic sample medical datasets (synthetic or de-identified).
    - Test users for UI/UX flows and proof generation.
    - IRB / Ethics consultants if simulating real-world deployments or designing for clinical settings.
